核心特性
1. 智能属性系统 (Smart Attributes)

•修饰器链 (Modifier Chain)：参考UE的GAS系统，实现了基于优先级的属性修改器。

•四则运算支持：支持 AddBase(基础加)、Add(加值)、Multiply(乘算)、Override(覆盖) 四种运算模式。

•无冲突叠加：无论加点、穿装备还是中Buff，系统都能按照既定优先级准确计算最终属性。

2. 高扩展性技能系统 (Skill System)

•原型模式 (Prototype)：所有技能基于原型字典存储，运行时通过复制构造函数实例化，避免污染原始数据。

•事件驱动：技能由多个“技能事件包”组成（如：造成伤害、回复能量、施加状态），通过Lambda闭包动态绑定施法者属性。

•链式配置：在C#代码中支持链式调用配置技能，极大提升开发效率。

3. 灵活的状态效果系统 (Buff/Debuff)

•三种叠加规则：支持 覆盖 (Overwrite)、升级 (LevelUp)、共存 (Independent)。

•两种衰减规则：支持 等级降低 (LevelDown)和 到期清除 (Clear)。

•回合管理：状态效果的添加、持续、衰减与移除全程由战斗状态机托管。

4. 数据驱动 (Data-Driven)

•JSON配置：地图节点、物品属性、商店货品、合成配方均通过JSON配置，无需硬编码。

•LitJson序列化：采用LitJson处理复杂的存档逻辑，支持字典与嵌套结构，完美保存动态生成的装备属性。

5. 性能优化

•低开销：纯UI交互，无复杂3D渲染，战斗场景帧率稳定在110FPS以上（i5-11400H）。

•事件中心：使用C#委托与事件解耦UI刷新与逻辑执行，避免频繁遍历查找。

技术架构

•引擎：Unity 2022.3 LTS

•语言：C# (.NET Framework)

•设计模式：

•单例模式 (Singleton)：用于全局管理器（数据、战斗、事件）。

•状态模式 (State)：战斗流程由有限状态机 (FSM) 驱动。

•原型模式 (Prototype)：技能与状态效果的实例化。

•观察者模式 (Observer)：事件中心驱动UI更新。

•数据持久化：LitJson (处理存档的序列化和反序列化)

游戏玩法

1.节点探索：在世界地图中随机遭遇敌人、商店、合成台。

2.角色养成：

•加点：自由分配属性点，支持一键重置。

•装备：获取并穿戴6个不同部位的装备，词条随机生成。

•技能：消耗金币升级技能，构建独特的战斗流派。

3.回合战斗：

•根据速度属性决定出手顺序。

•管理技能点与能量资源。

•利用状态效果（Buff/Debuff）扭转战局。

4.失败惩罚：遵循Roguelike精神，战斗失败将清空当前存档（无局外成长）。
